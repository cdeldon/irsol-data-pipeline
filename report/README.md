# IRSOL Solar Data Management System — Codebase Overview

**Audience:** Python developers building data pipelines for solar spectropolarimetric observations.
**As of:** March 2026.
**No prior knowledge of PHP is required** to use this document — PHP is treated as a black box; what matters is the data shapes and filesystem conventions the system relies on.

---

## What this system does

IRSOL (Istituto Ricerche Solari Aldo e Cele Daccò) operates solar telescopes that produce spectropolarimetric measurements. This system manages the full lifecycle of that data — from raw instrument files on an internal file server all the way to public scientific data products with DOIs and SVO registration.

The lifecycle in one sentence:
> Raw `.dat` / `.z3bd` files on **Sirius** → Python pipeline → **FITS files** → publication to **Zenodo**, **SVO**, **IrsolDB** → public browsing and download portal.

---

## Two services

The codebase contains two independent PHP web applications that communicate over a private REST API:

| Service | Directory | Purpose |
|---|---|---|
| **brightness-main** | `brightness-main/` | **Public-facing portal.** Any registered user can browse observation metadata filtered by year/wavelength/target, request FITS files, and download them. |
| **contrast-main** | `contrast-main/` | **Internal publisher portal.** IRSOL staff discover raw data on Sirius, trigger FITS generation, assign DOIs, publish to SVO and IrsolDB, and push the public catalogue to brightness-main. |

Each service has its own MySQL database. They run in separate Docker containers. Communication between them is HTTPS with mutual TLS (client certificates).

## Key vocabulary

| Term | Meaning |
|---|---|
| **Observation** | A named data-collection session, identified by a short code like `240317` (approximately YYMMDD). Groups one or more measurements taken on the same day. |
| **Measurement** | A single instrument run within an observation. Named like `n5140m1`, `nw5140m1`, `4607_m1`. This is the **atomic unit** — one FITS file per measurement. |
| **Sirius** | Internal SSH-accessible Linux file server where all raw and reduced instrument files are stored. Never directly accessible from the public internet. |
| **FITS file** | The main science data product. Multi-extension FITS holding calibrated Stokes I, Q, U, V arrays plus SOLARNET-compliant headers. Generated by `datapipeline.py`. |
| **IrsolDB** | External MySQL database at `db.irsol.ch`. Authoritative metadata registry (wavelength, observer, target, dates). Read by both services; written to via SSH script. |
| **SVO** | Solar Virtual Observatory (SolarNet, `solarnet.oma.be`). Federated registry of solar data. FITS files and thumbnails are registered here. |
| **DOI** | Digital Object Identifier, created via the Zenodo REST API (`zenodo.org`). Assigned to one or more FITS files grouped by the publisher. |
| **Quicklook image** | A `.jpg` preview of the spectral slit data, served from `db.irsol.ch`. |
| **Context image** | A `.jpg` showing the solar disk with the slit position highlighted, served from `db.irsol.ch`. |

## Telescope and instrument context

Two telescopes currently feed data into this system:

| Identifier | Telescope |
|---|---|
| `irsol` | IRSOL ZIMPOL (Zurich Imaging Polarimeter) |
| `gregor` | GREGOR ZIMPOL |

The instrument records polarimetric spectra of the solar atmosphere. Each measurement captures Stokes I, Q, U, V along the spectrograph slit at a wavelength around 5140 Å or 5160 Å (iron and magnesium lines). The Python pipeline performs wavelength calibration, slit inclination correction, and packages the data into FITS.
